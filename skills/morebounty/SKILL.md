# SRC漏洞挖掘增强流

只做六件事：维护上下文、选高价值面、用字典补覆盖、打关键漏洞、绕过阻断、证明真实影响。

## 硬规则

- 只处理授权SRC范围内资产；不越界扫无关IP、账号、云资源。
- 一次只聚焦一个目标、一个高价值功能、1-2类漏洞。
- WAF/CDN只是阻断层，不是漏洞根因；绕过只为证明已授权范围内的真实影响。
- 不做WAF过载、DoS、高并发压测；竞争条件只对自己账号和低风险动作最小验证。
- 不追低价值噪音：安全头、弱TLS、版本泄露、开放跳转单点、DNS-only SSRF、GraphQL introspection单点。
- 不写“可能、潜在、理论上”。能证明就推进，不能证明就放弃或降级。
- 需要命令时，只给当前路径的命令，不给工具全家桶。
- 人主导策略，AI只放大执行：枚举、归纳、脚本、报告初稿可以交给AI；范围判断、业务影响、是否提交由人确认。
- AI候选默认是待验证项；没过验证门不写成漏洞。

## 路径选择

```text
新目标 / 泛域名 / 不熟悉业务 -> 广扫
已有登录、接口、功能点 -> 深挖
已有漏洞现象 -> 验证
Payload被拦但业务可疑 -> 绕过
低危但可扩展 -> 链接
长期目标 / 信息超过一屏 -> 先建项目记忆卡
20分钟无进展 -> 换端点 / 换漏洞类 / 换子域
```

目标句式：

```text
我测试 [功能/接口]，目标是证明 [越权读/越权写/账号接管/资金损失/RCE/敏感数据泄露]。
```

## 广扫采集顺序

先建最小可用地图，不做无限枚举。

1. 资产：主站、管理后台、API域、移动端域、旧版本、CDN后源站线索、测试/预发环境。
2. 身份：匿名、普通用户、企业/租户用户、管理员、邀请用户、客服/审核后台。
3. 输入：URL、query、JSON、header、cookie、文件名、导入文件、模板、Markdown、富文本、WebSocket消息。
4. 输出：HTML、属性、JS、PDF、邮件、导出文件、日志、后台审核页、移动端返回、异步任务。
5. 技术：Server、框架、cookie名、JS包、source map、API文档、错误栈、WAF/CDN响应差异。

## 上下文与记录

长期目标先维护项目记忆卡；需要模板时读取 `references/project-memory.md`。

- 每次开始先读范围、账号角色、速率限制、已测攻击面、当前发现、已排除项。
- 新线索写成具体端点/功能，不写“测试XSS/SQL注入”这种泛建议。
- JS/API审计先产出端点地图，再按业务价值挑10个最值得深挖的具体点。
- 记录负向结果：端点、测试方式、排除原因；三天后不要重复浪费Token。
- 大目标拆会话：越权、认证、SSRF/文件、自由探索各管一块，共享同一份上下文文件。
- MCP/Skill少装：默认一个浏览器/抓包工具够用，除非当前证据证明需要更多。

## WAF/CDN绕过流程

先确认“被WAF拦”而不是业务失败。

1. 建基线：干净请求、明显恶意请求、边界字符请求；记录状态码、长度、拦截页、header、响应时间。
2. 定位检查点：payload放在query、body、JSON、header、cookie、文件名、路径，观察哪一段被拦。
3. 优先利用解析差异：WAF看到A，后端使用B；不要先随机换payload。
4. 绕过成功后必须证明后端真的执行/解析了payload；`200 OK`但内容被剥离不算。

| 类型 | 低噪声尝试 |
|---|---|
| 编码/规范化 | URL/双URL、大小写、Unicode、HTML实体、空白符、注释切分 |
| 参数名绕过 | 参数名前后空白、点/中划线/下划线差异、数组参数、框架自动归一化 |
| HPP/HPF | 重复参数、同义参数、把关键字拆到多个参数，确认后端取首个/末个/拼接/数组 |
| Content-Type | form、JSON、multipart、XML互换；文件字段、文件名、part header分别测试 |
| 方法/路径 | GET/POST/PUT/PATCH/DELETE、`X-HTTP-Method-Override`、`/./`、`//`、`;`路径参数、尾点/尾空格 |
| Header/IP | `X-Forwarded-For`、`X-Real-IP`、`Forwarded`、`True-Client-IP`；只验证信任边界，不伪造第三方身份 |
| 协议层 | chunked、HTTP/2、绝对URI请求行、前后端解析差异；有走私迹象才深入 |
| 语义替代 | SQL函数替代、无空格/无逗号写法、XSS事件/标签替代、命令拼接/变量展开 |

停止：连续3类解析差异无效、需要高流量、需要越界源站探测、或无法证明业务影响。

## 漏洞路线

### 越权 / IDOR / BOLA / BFLA / Mass Assignment

- 准备A、B两个普通账号；多租户场景再加A租户内低权用户。
- 用B的登录态访问A的对象ID；测GET、POST、PUT、PATCH、DELETE。
- 查ID位置：路径、query、JSON、header、cookie、GraphQL变量、嵌套对象、批量数组。
- 查功能级鉴权：普通用户调管理员接口、导出接口、审批接口、批量接口、旧版本接口。
- 查隐藏可写字段：`role`、`isAdmin`、`tenant_id`、`owner`、`status`、`price`、`verified`。
- 有A的数据返回、对象被修改、权限被提升、或跨租户数据变化才算有效。

### 认证 / 会话 / JWT / OAuth / SSO / MFA

- 跳步骤：未完成登录直接访问登录后接口；未完成MFA访问敏感接口。
- 复用token、reset token、invite token、旧session；测注销、改密、绑定邮箱后的失效。
- JWT先看`alg`、`kid`、JWKS来源、过期、aud/iss、role/tenant claim；不盲喷。
- OAuth/OIDC看`redirect_uri`精确匹配、state绑定、code复用、PKCE、账号绑定回调、开放跳转链。
- SAML/SSO看断言受众、签名覆盖、NameID/邮箱映射、租户归属。
- 找回密码重点看Host头、token泄露、token可复用、账号归属校验。
- “管理员能做”不算；“普通用户能做管理员动作”才算。

### 业务逻辑 / 竞争条件

- 先写清业务不变量：一次券只能用一次、余额不能为负、未付款不能发货、低权不能审批高权。
- 测改价、改数量、改币种、负数、跳支付、重放回调、重复退款、重复领奖、库存/额度扣减。
- 测状态机：跳步骤、回退状态、并行提交、旧订单重放、客户端字段覆盖服务端计算。
- 并发只对自己账号和低风险动作做最小验证；金钱、积分、权限、库存、额度真实变化才算。

### SSRF / URL处理 / Webhook / PDF / 截图

- 先用唯一OOB域名证明服务端发起请求；每个参数不同标签，能定位触发点。
- 查URL解析差异：重定向链、userinfo、IPv6、十进制/八进制IP、短域、DNS重绑定、大小写scheme。
- 查协议/位置：HTTP(S)、gopher/file只在规则允许时验证；header、JSON、文件元数据、HTML渲染资源都算入口。
- DNS-only只算线索，必须继续证明内部服务、云元数据、敏感数据或后台动作。

### 文件上传 / 导入 / 处理链

- 分四段测：Accept校验、Store命名/权限、Process解析/转码/解压、Serve预览/下载/分享。
- 测扩展名、Content-Type、魔数、双后缀、大小写、尾点、SVG/HTML、OOXML、EXIF、压缩包路径。
- 关注存储型XSS、路径穿越、覆盖文件、解析器SSRF/XXE、反序列化、命令注入、可执行落地。
- 文件名、元数据、导出记录、后台审核页都当作存储输入。
- “能上传奇怪后缀”不算，必须有执行、读取、覆盖、跨租户访问或可信上下文执行。

### 路径穿越 / LFI / 下载预览

- 优先测下载、预览、模板、语言包、静态别名、压缩包解压、日志查看、报表导出。
- 先用低风险文件证明读取：Linux `/etc/hostname`/`/etc/passwd`，Windows `win.ini`，Java `WEB-INF/web.xml`。
- 再找影响：环境变量、应用配置、数据库连接、云凭据、源码、日志路径、session文件。
- 绕过顺序：`../`、URL编码、双编码、反斜杠、冗余点斜杠、绝对路径、Tomcat `;`路径参数、Nginx alias错配。
- LFI到RCE只做最小无害证明；读到敏感配置即可优先报告。

### GraphQL / API配置

- 枚举query、mutation、参数、对象ID、批量接口、alias、fragment、隐藏字段。
- 用B token访问A对象；mutation优先；测试字段级、对象级、函数级鉴权。
- 查批量请求、深层嵌套、旧schema、移动端schema、admin示例、`additionalProperties`和未文档字段。
- introspection本身不报，除非带出越权、敏感字段或可利用mutation。

### XSS / DOM / 存储HTML

- 先判上下文：HTML文本、属性、JS字符串、URL、SVG/XML、Markdown、富文本、PDF、邮件。
- 优先后台、客服、审核、富文本、Markdown、SVG、文件名、日志字段、导出预览。
- 盲打/存储型用OOB证明；编码后的payload不算执行。
- 绕过WAF先换上下文和语义：事件、标签、实体、大小写、协议、DOM clobbering、二次渲染。
- 必须链到后台动作、会话/token、CSRF、OAuth、租户数据或可信上下文。

### 注入 / RCE / 解释器类

- 搜索、筛选、排序、模板、表达式、导入解析、报错栈、异步任务、命令行工具优先。
- 覆盖SQL、NoSQL、SSTI、XXE、XSLT、命令注入、反序列化、JNDI、表达式语言、原型污染。
- 先做无害差异测试：状态、响应体、时间、错误、OOB；时间型至少多次交错对照。
- 关注非参数化位置：排序字段、表/列名、JSON key、Mongo操作符、GraphQL参数、header、cookie。
- RCE只做最小无害证明，证明后停止。

### 请求走私 / 缓存 / CORS / CSRF / WebSocket

- 只有出现前后端解析差异、代理/CDN、HTTP/2降级、异常连接复用时才测请求走私。
- 缓存漏洞必须证明无攻击头的干净请求也拿到投毒/欺骗结果。
- CORS只报可带凭据读取敏感数据；单纯反射Origin无敏感读不报。
- CSRF必须证明敏感状态改变且缺少有效token/SameSite/二次校验；JSON和multipart都测。
- WebSocket重点看握手鉴权、消息级越权、重放、跨站WS、订阅他人频道。

### 源码泄露 / 密钥 / 供应链

- 优先查`.env`、`.git`、source map、swagger/openapi、build-info、debug、actuator、备份包。
- 从源码里反推隐藏接口、鉴权逻辑、对象ID来源、签名算法、旧API、队列/webhook密钥。
- 密钥只用只读接口验证权限范围；不做写入、删除、横向扩展。
- 报告密钥能访问什么，不报告“疑似密钥字符串”。
- 内部包名、私有registry、CI配置泄露时，再评估依赖混淆/供应链风险。

## 链接方法

确认A后，用20分钟找B。

| A | 找B | 目标 |
|---|---|---|
| IDOR读 | 同模块写/删/管理接口、批量接口 | 越权修改、提权 |
| BFLA | admin函数、旧版本、移动端接口 | 普通用户执行高权动作 |
| Mass assignment | role/tenant/status/price字段 | 提权、跨租户、资金影响 |
| 开放跳转 | OAuth `redirect_uri` / 登录回跳 | ATO |
| SSRF回连 | 内部服务/元数据/截图资源 | 敏感数据或云权限 |
| 存储XSS | 后台查看/CSRF动作/OAuth授权页 | 提权 |
| source map | 隐藏API/密钥/对象ID算法 | 越权或密钥影响 |
| GraphQL schema | mutation鉴权/批量alias | 批量泄露/修改 |
| 文件上传 | SVG/解析器/路径/异步处理器 | XSS、SSRF、覆盖、RCE |
| 路径穿越 | 配置/源码/日志/session | 凭据泄露、LFI链 |
| Host反射 | 重置邮件/缓存/绝对链接 | ATO或缓存投毒 |
| WAF绕过 | 同过滤器复用的接口 | 扩展到真实漏洞影响 |
| 请求走私 | 后台路由/缓存/鉴权前置 | 绕鉴权、缓存投毒 |
| 竞争条件 | 钱包/优惠券/退款/库存 | 资金影响 |

停止：20分钟无B、连续3个B失败、需要3个以上苛刻前提。

## 验证门

候选漏洞默认放“待验证”，全部通过后才进入报告。

| Gate | 要求 |
|---|---|
| 0 | 有精确请求、PoC或最小复现步骤 |
| 1 | 换干净登录态/重置状态后仍可复现 |
| 2 | 普通攻击者权限可触发，且在SRC范围内 |
| 3 | 证明真实业务影响，不只是报错、反射或状态码差异 |
| 4 | 写清已排除的误报解释：缓存、自己数据、前端假象、WAF拦截页、管理员权限等 |

任一失败：留在待验证或已排除，不提交。

## 快速验证

提交前只问：

```text
1. 是否有可复现的请求？
2. 是否在SRC范围内？
3. 普通攻击者权限是否可达？
4. 是否证明了真实影响？
5. WAF/CDN绕过是否证明后端真实处理？
6. 是否不是常见无效项，或已经完成链路？
```

任一失败：杀掉、降级或继续补证。

## 误报过滤

- 只看响应码不够，必须看响应体、状态变化、对象归属或OOB证据。
- 反射测试用随机标记，不用test、evil、payload。
- 盲SSRF/XSS/XXE/RCE必须有OOB。
- WAF返回200但payload被剥离、转义、沙箱执行，不算绕过。
- 缓存投毒必须证明无攻击头的干净请求也拿到投毒内容。
- IDOR必须证明攻击者原本无权访问。
- CORS必须证明带凭据读到敏感响应。
- 密钥必须证明只读可用范围。

## 简约报告

报告去AI味：短句、具体接口、具体影响、具体证据；删掉寒暄、套话和空泛严重性。

```markdown
# [漏洞类型] [接口/功能] 导致 [具体影响]

## 影响
[攻击者可以做什么，影响哪些用户/数据/资产。]

## 范围
- 目标:
- 接口:
- 权限:

## 复现
1. 准备:
2. 请求:
3. 结果:
4. 若有WAF/CDN: 绕过前后差异与后端处理证据:

## 证据
[响应片段/截图/OOB记录/状态变化，敏感信息打码。]

## 已排除误报
[缓存/自身数据/权限差异/前端假象/WAF拦截等排除结果。]

## 修复

```
