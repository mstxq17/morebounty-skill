# SRC漏洞挖掘核心流

只做四件事：选高价值面、打关键漏洞、证明真实影响、写简报。

## 硬规则

- 只处理授权SRC范围内资产。
- 一次只聚焦一个目标、一个高价值功能、1-2类漏洞。
- 不追低价值噪音：安全头、弱TLS、版本泄露、开放跳转单点、DNS-only SSRF、GraphQL introspection单点。
- 不写“可能、潜在、理论上”。能证明就推进，不能证明就放弃或降级。
- 需要命令时，只给当前路径的命令，不给工具全家桶。

## 固定输出

开始、切换目标、卡住时，先输出：

```text
目标:
模式: 广扫 / 深挖 / 验证 / 链接
核心资产:
当前漏洞类:
P1:
P2:
跳过:
下一步3个动作:
需要的证据:
停止条件:
```

## 路径选择

```text
新目标 / 泛域名 / 不熟悉业务 -> 广扫
已有登录、接口、功能点 -> 深挖
已有漏洞现象 -> 验证
低危但可扩展 -> 链接
20分钟无进展 -> 换端点 / 换漏洞类 / 换子域
```

目标句式：

```text
我测试 [功能/接口]，目标是证明 [越权读/越权写/账号接管/资金损失/RCE/敏感数据泄露]。
```

## 攻击面排序

优先级只看价值和可打性。

| 信号 | 优先级 |
|---|---|
| 钱、订单、账单、积分、优惠券、提现、退款 | P1 |
| 用户数据、私信、文件、导出、企业/租户数据 | P1 |
| 登录、注册、找回密码、邀请、MFA、OAuth、SSO | P1 |
| `/api/`、移动端API、旧版本API、GraphQL、OpenAPI | P1 |
| 文件上传、导入、Webhook、URL预览、PDF/截图生成 | P1 |
| `.env`、`.git`、source map、swagger、debug、actuator | P1 |
| 静态官网、帮助文档、纯CDN、全403、无登录无接口 | 跳过 |

## 漏洞路线

### 越权 / IDOR / BOLA

- 准备A、B两个普通账号。
- 用B的登录态访问A的对象ID。
- 测GET、POST、PUT、PATCH、DELETE。
- 扩展同模块、旧版本API、移动端API、GraphQL mutation。
- 有A的数据返回或对象被修改才算有效。

### 认证 / 会话 / MFA / 找回密码

- 跳步骤：未完成登录直接访问登录后接口。
- 复用token、reset token、invite token、旧session。
- 比较Web、移动端、旧接口的鉴权差异。
- 找回密码重点看Host头、token泄露、token可复用、账号归属校验。
- “管理员能做”不算；“普通用户能做管理员动作”才算。

### 业务逻辑 / 竞争条件

- 先写清业务不变量：一次券只能用一次、余额不能为负、未付款不能发货。
- 测改价、改数量、改币种、跳支付、重放回调、重复退款。
- 并发只对自己账号和低风险动作做最小验证。
- 金钱、积分、权限、库存、额度发生真实变化才算。

### SSRF / URL处理 / Webhook / PDF

- 先用唯一OOB域名证明服务端发起请求。
- 每个参数用不同标签，能定位触发点。
- 回显URL、报错差异、单次延迟不算SSRF。
- DNS-only只算线索，必须继续证明内部服务、云元数据或敏感数据。

### 文件上传 / 导入

- 测扩展名、Content-Type、魔数、SVG/HTML渲染、压缩包解压、文件名路径。
- 关注存储型XSS、路径穿越、覆盖文件、解析器SSRF/XXE、可执行落地。
- “能上传奇怪后缀”不算，必须有执行、读取、覆盖或越权影响。

### GraphQL / API配置

- 枚举query、mutation、参数、对象ID。
- 用B token访问A对象；mutation优先。
- 测批量请求、alias、隐藏字段、旧版本接口、字段级鉴权。
- introspection本身不报，除非带出越权、敏感字段或可利用mutation。

### XSS / DOM / 存储HTML

- 优先后台、客服、审核、富文本、Markdown、SVG、文件名、日志字段。
- 编码后的payload不算执行。
- 盲打/存储型用OOB证明。
- 必须链到后台动作、会话/token、CSRF、OAuth、租户数据或可信上下文。

### 注入 / RCE

- 搜索、筛选、排序、模板、表达式、导入解析、报错栈优先。
- 先做无害差异测试：状态、响应体、时间、错误。
- 时间型至少多次交错对照，单次慢响应不算。
- RCE只做最小无害证明，证明后停止。

### 源码泄露 / 密钥

- 优先查`.env`、`.git`、source map、swagger/openapi、build-info、debug。
- 从源码里反推隐藏接口、鉴权逻辑、对象ID来源。
- 密钥只用只读接口验证权限范围。
- 报告密钥能访问什么，不报告“疑似密钥字符串”。

## 链接方法

确认A后，用20分钟找B。

| A | 找B | 目标 |
|---|---|---|
| IDOR读 | 同模块写/删/管理接口 | 越权修改、提权 |
| 开放跳转 | OAuth redirect_uri | ATO |
| SSRF回连 | 内部服务/元数据 | 敏感数据或云权限 |
| 存储XSS | 后台查看/CSRF动作 | 提权 |
| source map | 隐藏API/密钥 | 越权或密钥影响 |
| GraphQL schema | mutation鉴权 | 批量泄露/修改 |
| 文件上传 | SVG/解析器/路径 | XSS、SSRF、覆盖 |
| Host反射 | 重置邮件/缓存 | ATO或缓存投毒 |
| 竞争条件 | 钱包/优惠券/退款 | 资金影响 |

停止：20分钟无B、连续3个B失败、需要3个以上苛刻前提。

## 快速验证

提交前只问：

```text
1. 是否有可复现的请求？
2. 是否在SRC范围内？
3. 普通攻击者权限是否可达？
4. 是否证明了真实影响？
5. 是否不是常见无效项，或已经完成链路？
```

任一失败：杀掉、降级或继续补证。

## 误报过滤

- 只看响应码不够，必须看响应体差异。
- 反射测试用随机标记，不用test、evil、payload。
- 盲SSRF/XSS/XXE/RCE必须有OOB。
- 缓存投毒必须证明无攻击头的干净请求也拿到投毒内容。
- IDOR必须证明攻击者原本无权访问。
- 密钥必须证明只读可用范围。
